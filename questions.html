1. Now that you've had a chance to implement it, are there different features you would have prioritized, knowing what you know now?

Understanding/Planning

With a firmer grasp of the optics concepts, I think I would do a better job of planning out how the code should look like. I got to understand optics better by writing out code and trying to model it. Though, the limits of my code suggest there's still more for me to understand about optics. A better understanding and better planning would definitely have helped me optimize my time.

Instance Mode

I personally dislike the syntax style of adding p5 prefixes everywhere. For this project's scope, I felt prioritizing readable js would be reasonable. However, this would definitely be an important thing to prioritize from the start for next time.

Reworking Normal Calculation

I definitely would go to rework how the normal is calculated for a line. I feel this got tricky for mirrors away from the origin. Ideally, the normal would be able to work for any quadrant and at any rotation. With more time, I'd look into using dot products similar to what was done here. With more testing, figuring out the normal for any given line should allow for the reflections to be easily calculated by reflecting on the normal I used this formula for calculating the normal but did try out a bunch of other different ways.

Reworking Light Stoppage

A tricky problem was in stopping light from continuing past a mirror/wall. Due to using a canvas, we cannot truly 'undraw' what's already there, so I implemented a hacky solution of just covering the continuous sections with a black line matching the background. I'd prefer to implement a canvas state control of some sort to provide a cleaner way of stopping light and its reflections.

Other

  • Adding Drag and Drop
  • Adjustable rotation in canvas
  • Responsiveness
  • Accessibility

3. What parts of your interactive could be useful in other areas of STEM? Give a few examples.

A possible area could be understanding the trajectory in a game of pool. An overhead view offers a 2D view that can be similarly modelled as optics. Other similar bouncing trajectory concepts could also be done.

Visualizing the math required to calculate the reflections could be useful in teaching geometry or trigonometry. Concepts like normals or deltas could be taught interactively in different ways.
